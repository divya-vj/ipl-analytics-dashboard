## 🎯 Overview

Raw cricket data is useless without the right questions. This dashboard 
takes 260,920 ball-by-ball IPL deliveries and turns them into answers:

- Which venues favour teams that bat first?
- Has IPL scoring increased over 17 seasons?
- Who is the most economical bowler with at least 300 balls bowled?
- Who was the top scorer in each season?

Built to demonstrate end-to-end data skills — from raw CSV to deployed 
interactive web application.

**Dataset:** 1,095 matches · 260,920 deliveries · 17 seasons (2007–2024)

## 🛠️ Tech Stack

| Technology | Purpose |
|---|---|
| Python 3.10 | Core language |
| SQLite | Serverless database — single file, zero configuration |
| pandas | CSV to DataFrame loading, read_sql_query integration |
| Streamlit | Interactive web app — zero HTML or JavaScript needed |
| Plotly Express | Interactive charts with hover, zoom, and filter |
| Streamlit Cloud | Free deployment, public URL from GitHub |

**Why SQLite over PostgreSQL?** This dataset is static — it does not change while the app runs. SQLite needs no server, no credentials, no configuration. PostgreSQL would add infrastructure complexity with zero benefit here.

**Why Streamlit over Flask?** Flask requires HTML templates, CSS, and JavaScript. Streamlit converts Python directly to a web app. For a data dashboard, Streamlit is the professional standard.

## 🔍 SQL Concepts Used

**10 queries total.** Concepts covered:
```sql
-- JOIN + Subquery: Average first innings score per season
SELECT m.season, ROUND(AVG(t.innings_total), 0) AS avg_score
FROM matches m
JOIN (
    SELECT match_id, SUM(total_runs) AS innings_total
    FROM deliveries WHERE inning = 1 GROUP BY match_id
) t ON m.id = t.match_id
GROUP BY m.season ORDER BY m.season;

-- Window Function: Top scorer per season
SELECT season, batter, season_runs FROM (
    SELECT m.season, d.batter,
           SUM(d.batsman_runs) AS season_runs,
           ROW_NUMBER() OVER (
               PARTITION BY m.season
               ORDER BY SUM(d.batsman_runs) DESC
           ) AS rank_in_season
    FROM deliveries d JOIN matches m ON d.match_id = m.id
    GROUP BY m.season, d.batter
) ranked WHERE rank_in_season = 1;

-- CASE WHEN: Count sixes per batsman
SELECT batter,
       SUM(CASE WHEN batsman_runs = 6 THEN 1 ELSE 0 END) AS total_sixes
FROM deliveries GROUP BY batter ORDER BY total_sixes DESC;

-- HAVING: Filter bowlers with minimum sample size
SELECT bowler, ROUND(SUM(total_runs)*6.0/COUNT(*), 2) AS economy
FROM deliveries
GROUP BY bowler HAVING COUNT(*) >= 300
ORDER BY economy ASC LIMIT 10;
```

## ⚙️ Installation
```bash
# Clone the repo
git clone https://github.com/divya-vj/ipl-analytics-dashboard.git
cd ipl-analytics-dashboard

# Install dependencies
pip install -r requirements.txt

# Build the database (run once)
python db_setup.py

# Launch the dashboard
streamlit run app.py
# Opens at http://localhost:8501
```

**Note:** Download the IPL dataset from Kaggle and place 
`matches.csv` and `deliveries.csv` in the `data/` folder before 
running `db_setup.py`.

**Dataset link:** [IPL Complete Dataset — Kaggle](https://www.kaggle.com/datasets/patrickb1912/ipl-complete-dataset-20082020)
